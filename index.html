SPARTA System Performance Analysis Ready To Advance

Sparta is a load-testing tool running your favorite test-tools from the cloud.

  • scriptable load-tests using a simple DSL
  • fully automated setup of cloud instances at providers of your choice
  • easy integration of load-testing tools you already use
  • graphical presentation of testing results

Install dependencies with bundler: bundle install

Getting Started

To get started you will need an account at one of the supported cloud computing providers:

  • AmazonEC2
  • Rackspace
  • tbd.

Adding the API credentials you received from the provider of your choice to the local configuration config/credentials.yaml or ~/.netrc will make your scripts cleaner and more portable. Nevertheless, credentials can also be included in scripts where needed to override default credentials in both files above.

Installing Sparta

To install sparta, simply install the gem.

gem install sparta

Building Blocks

Warriors

Warriors are the basic testing blocks. They handle both the connection, i.e. the instance they are using to run the test, as well as the weapon, which is a means of running a test. The coordination of those blocks is done by the warrior. The API is quite simple, for instance, this snippet will create a new warrior that performs an ApacheBenchmark test from an EC2 instance.

That will launch a micro instance, equip the instance with the necessary tools and execute the given tests. The results will be collected ( in form of a CSV-Logfile ) and provided for further analysis of the data.

Since warriors are a bit weak to test bigger setups, Squads are a way to configure and control groups of warriors to do something.

Squads

Squads are a thin management layer above warriors. They provide no testing functionality that is not existing in the warriors or weapons, but ae extremely convinient when it comes to assemble larger groups of warriors and coordinate their attacks.

Squads are WIP and a example will be provided as soon as they are ready for prime time.
